## Архитектура локализации

### Структура файлов

```
PyPath/
├── backend/
│   └── app/
│       └── core/
│           └── locales.py          # Переводы курсов и миссий
├── constants.tsx                    # UI переводы и управление языком
├── components/
│   ├── Dashboard.tsx               # Локальные переводы компонента
│   ├── Courses.tsx                 # Локальные переводы компонента
│   └── Editor.tsx                  # Динамическая генерация контента
├── index.html                      # Конфигурация шрифтов
└── styles.css                      # Типографика
```

### Поддерживаемые языки

- **Русский (ru)** - язык по умолчанию
- **Казахский (kz)** - полная поддержка с Cyrillic Extended

### Как работает локализация

1. Пользователь выбирает язык в настройках
2. Язык сохраняется в `localStorage` под ключом `app-language`
3. Константа `APP_LANGUAGE` обновляется через `setAppLanguage()`
4. API запросы отправляются с заголовком `Accept-Language`
5. Бэкенд возвращает локализованные данные
6. Фронтенд отображает данные на выбранном языке

## Добавление переводов курсов

### Шаг 1: Откройте файл локализации

```bash
backend/app/core/locales.py
```

### Шаг 2: Добавьте перевод в COURSE_TRANSLATIONS

```python
COURSE_TRANSLATIONS = {
    "kz": {
        # Существующие переводы...
        
        # Новый курс
        19: {
            "title": "Тарау 19: Название на казахском",
            "description": "Описание курса на казахском языке.",
            "section": "Секция на казахском",
        },
    }
}
```

### Структура перевода курса

| Поле | Обязательно | Описание |
|------|-------------|----------|
| `title` | ✅ Да | Название курса |
| `description` | ✅ Да | Описание курса |
| `section` | ⚠️ Рекомендуется | Секция/категория курса |

### Пример полного перевода

```python
17: {
    "title": "Тарау 17: 8 сыныпқа арналған Python",
    "description": "8 сынып негіздерін бекіту: шарттар, циклдер және шағын алгоритмдік тапсырмалар.",
    "section": "8 сынып: негізгі дағдылар",
},
```

### Структура UI переводов

UI переводы организованы по секциям:

- `sidebar` - боковая панель
- `header` - шапка сайта
- `app` - общие сообщения приложения
- `courses` - страница курсов
- `dashboard` - главная страница
- `settings` - настройки
- `profile` - профиль
- `editor` - редактор кода
- `achievements` - достижения
- `leaderboard` - рейтинг
- `community` - сообщество
- `aiChat` - чат с AI
- `aiChatPage` - страница чата

### Пример добавления новой секции

```typescript
const KZ_TEXT_OVERRIDES = {

        startButton: 'Бастау',
```

### Использование переводов в компонентах

#### Вариант 1: Через UI_TEXTS (рекомендуется)

```typescript
import { UI_TEXTS, APP_LANGUAGE } from '../constants';

const MyComponent = () => {
    const text = UI_TEXTS?.newFeature ?? {};
    
    return (
        <div>
            <h1>{text.title}</h1>
            <p>{text.description}</p>
            <button>{text.startButton}</button>
        </div>
    );
};
```

#### Вариант 2: Локальные переводы (для простых случаев)

```typescript
import { APP_LANGUAGE } from '../constants';

const MyComponent = () => {
    const isKz = APP_LANGUAGE === 'kz';
    const lt = {
        title: isKz ? 'Заголовок KZ' : 'Заголовок RU',
        button: isKz ? 'Кнопка KZ' : 'Кнопка RU',
    };
    
    return (
        <div>
            <h1>{lt.title}</h1>
            <button>{lt.button}</button>
        </div>
    );
};
```

## Тестирование переводов

### Автоматическая проверка

Запустите скрипт проверки переводов:

```bash
node check-translations.js
```

Скрипт проверит:
- ✅ Наличие переводов для всех курсов
- ✅ Наличие переводов для миссий
- ✅ Количество UI переводов
- ✅ Конфигурацию шрифтов
- ✅ CSS правила для типографики

### Ручное тестирование

#### 1. Проверка переключения языка

1. Откройте приложение
2. Перейдите в Настройки
3. Переключите язык на казахский
4. Проверьте, что все элементы интерфейса изменились

#### 2. Проверка типографики

1. Откройте страницу с казахским текстом
2. Проверьте отображение специальных символов: ә, і, ң, ғ, ү, ұ, қ, ө, һ
3. Убедитесь, что все символы одинакового размера и толщины

#### 3. Проверка на мобильных устройствах

1. Откройте DevTools (F12)
2. Включите режим эмуляции мобильного устройства
3. Проверьте перенос длинных слов
4. Убедитесь в отсутствии горизонтальной прокрутки

#### 4. Проверка контента уроков

1. Откройте любой урок
2. Проверьте, что теория на казахском
3. Проверьте, что подсказки на казахском
4. Проверьте, что сообщения об ошибках на казахском

### Тестовые данные

Используйте файл `TYPOGRAPHY_TEST.md` для проверки отображения всех казахских символов.

## Лучшие практики

### 1. Единообразие терминологии

Используйте одинаковые термины во всех переводах:

| Русский | Казахский | Примечание |
|---------|-----------|------------|
| Переменная | Айнымалы | Не "өзгермелі" |
| Функция | Функция | Транслитерация |
| Цикл | Цикл | Транслитерация |
| Условие | Шарт | Не "жағдай" |
| Список | Тізім | Не "тізімдеме" |

### 2. Длина текста

Казахские переводы часто длиннее русских. Учитывайте это при дизайне:

```typescript
// ❌ Плохо: текст может не поместиться
<button className="px-2 py-1 text-xs">
    {text.veryLongButtonText}
</button>

// ✅ Хорошо: достаточно места
<button className="px-4 py-2 text-sm">
    {text.veryLongButtonText}
</button>
```

### 3. Контекст

Всегда предоставляйте контекст для переводчиков:

```python
# ❌ Плохо: непонятно, что это
"title": "Старт"

# ✅ Хорошо: понятен контекст
"title": "Тарау 1: Python-дағы алғашқы қадамдар"  # Название первой главы курса
```

### 4. Плюрализация

Учитывайте правила множественного числа:

```typescript
// Казахский язык
const getPracticeWord = (count: number) => {
    return 'практика'; // В казахском нет изменения по числам
};

// Русский язык
const getPracticeWord = (count: number) => {
    if (count === 1) return 'практика';
    if (count >= 2 && count <= 4) return 'практики';
    return 'практик';
};
```

### 5. Форматирование

Сохраняйте форматирование в переводах:

```python
# ✅ Хорошо: сохранены переносы строк и отступы
"hints": [
    "Цикл ішінде print арқылы шығару қосыңыз.\nfor port in ports:\n    print('Портты тексереміз', port)"
]
```

### 6. Специальные символы

Всегда используйте правильные казахские символы:

```
✅ Правильно: ә, і, ң, ғ, ү, ұ, қ, ө, һ
❌ Неправильно: а, и, н, г, у, у, к, о, х
```

### 7. Тестирование

Всегда тестируйте переводы в реальном интерфейсе:

1. Проверьте отображение на разных экранах
2. Проверьте перенос длинных слов
3. Проверьте типографику
4. Проверьте контекст использования

## Частые ошибки

### ❌ Ошибка 1: Неправильные символы

```python
# Неправильно
"title": "Тарау 1: Python-дагы алгашкы кадамдар"  # Использованы русские буквы

# Правильно
"title": "Тарау 1: Python-дағы алғашқы қадамдар"  # Использованы казахские буквы
```

### ❌ Ошибка 2: Отсутствие контекста

```typescript
// Неправильно
const text = isKz ? 'Бастау' : 'Старт';  // Непонятно, что это

// Правильно
const startButtonText = isKz ? 'Оқуды бастау' : 'Начать обучение';
```

### ❌ Ошибка 3: Жестко заданные строки

```typescript
// Неправильно
<h1>Сіздің оқу аймағыңыз</h1>  // Жестко задано на казахском

// Правильно
<h1>{text.baseTitle || lt.zoneTitle}</h1>  // Динамически из переводов
```

## Добавление нового языка

Если нужно добавить поддержку нового языка (например, английского):

### 1. Обновите типы

```typescript
// constants.tsx
export type AppLanguage = 'ru' | 'kz' | 'en';
```

### 2. Добавьте переводы на бэкенде

```python
# backend/app/core/locales.py
COURSE_TRANSLATIONS = {
    "kz": { ... },
    "en": {
        1: {
            "title": "Chapter 1: First Steps in Python",
            "description": "...",
            "section": "...",
        },
    }
}
```

### 3. Добавьте UI переводы

```typescript
// constants.tsx
const EN_TEXT_OVERRIDES = {
    sidebar: {
        logoLine1: 'Py',
        logoLine2: 'Path',
    },
    // ... остальные переводы
};
```

### 4. Обновите функцию локализации

```typescript
const localizeTexts = (texts: any) => {
    const base = isObject(texts) ? texts : {};
    if (APP_LANGUAGE === 'kz') return deepMerge(base, KZ_TEXT_OVERRIDES);
    if (APP_LANGUAGE === 'en') return deepMerge(base, EN_TEXT_OVERRIDES);
    return base;
};
```
